Overview

The Student Management System is a comprehensive web application built with Spring Boot and Thymeleaf, designed to manage educational institutions' core entities: Students, Parents, and Courses.

Interactive API Testing

This application provides a complete REST API for all features. You can explore and test all endpoints using our interactive Swagger UI documentation.

Technology Stack
  • Spring Boot Framework
  • Thymeleaf Template Engine
  • Spring Data JPA
  • Bootstrap CSS Framework
  • Springdoc OpenAPI (Swagger)
Key Benefits
  • Centralized student data management
  • Parent-student relationship tracking
  • Course enrollment management
  • Responsive web interface
  • Comprehensive REST API with documentation

Features Overview

Student Management

Comprehensive student information management system with full CRUD operations through both web interface and REST API.

All student operations are available via API endpoints. Test them in Swagger UI.
Functionality
  • Add new students with personal details
  • View student profiles and information
  • Update student records
  • Delete student entries
  • Search and filter students
Student Information
  • Full name and contact details
  • Date of birth and age calculation
  • Email and phone number
  • Address information
  • Enrollment date and status
Available Endpoints
Method Endpoint Description
GET /api/students Get all students (JSON)
GET /students Display all students (UI)
GET /students/new Show add student form
POST /api/students Create new student (API)
GET /students/{id}/edit Show edit student form
GET /api/students/{id} Get student by ID (API)
DELETE /api/students/{id} Delete student (API)

Parent Management

Complete parent information system with student relationship management through both web interface and REST API.

All parent operations are available via API endpoints. Test them in Swagger UI.
Functionality
  • Register new parent profiles
  • Link parents to student records
  • Manage parent contact information
  • Update parent details
  • Track parent-student relationships
Parent Information
  • Full name and relationship type
  • Contact phone and email
  • Address and emergency contacts
  • Occupation and workplace
  • Associated student records
Available Endpoints
Method Endpoint Description

POST /api/parents Create new parent (API)
GET /parents/{id}/edit Show edit parent form

Course Management

Comprehensive course management system with enrollment and scheduling capabilities through both web interface and REST API.

All course operations are available via API endpoints. Test them in Swagger UI.
Functionality
  • Create and manage course catalogs
  • Handle student enrollments
  • Track course capacity and availability
  • Manage course schedules
  • Generate enrollment reports
Course Information
  • Course name and description
  • Course code and credits
  • Instructor and schedule
  • Maximum enrollment capacity
  • Prerequisites and requirements
Available Endpoints
Method Endpoint Description

GET /courses/new Show add course form
POST /api/courses Create new course (API)
GET /courses/{id}/edit Show edit course form
GET /api/courses/{id} Get course by ID (API)

Error Handling
HTTP Status Error Type Description

API Testing with Swagger UI

Interactive API Documentation

Our application includes a comprehensive Swagger UI interface that allows you to:

  • Explore all available API endpoints
  • View request and response schemas
  • Execute API calls directly from your browser
  • Test all CRUD operations for students, parents, and courses
  • View authentication requirements (if applicable)
Open Swagger UI
Getting Started with API Testing
  1. Ensure the application is running on http://localhost:8080
  2. Click the button to open Swagger UI in a new tab
  3. Expand the controller sections to see available endpoints
  4. Click "Try it out" on any endpoint to test it
  5. Fill in required parameters and execute the request
  6. View the server response including status codes and data
Note: Changes made through the API will affect your database. Use with caution in production environments.
